Angular Route Guards

Guards help protect routes, control access, and improve application security in Angular.

Authentication vs Authorization

Authentication

Authentication verifies the identity of the user. Example: Logging into an application using username and password.

Authorization

Authorization decides what resources or pages the authenticated user can access.

Why Do We Use Guards?

✔️ Protect routes from unauthorized users

✔️ Redirect users to login page

✔️ Restrict access based on roles

✔️ Prevent lazy-loaded modules from downloading
