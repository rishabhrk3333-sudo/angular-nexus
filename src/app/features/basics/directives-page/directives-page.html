Angular Basics

Angular Directives

Directives are one of Angular’s most powerful features. They allow developers to manipulate the DOM, add dynamic behavior, and build reusable UI functionality.

⚡

What are Directives in Angular?

Directives extend and manipulate DOM behavior.

Directives are instructions in Angular that help manipulate the DOM and add behavior to existing elements or even create new elements.

They are a key feature of Angular used to create dynamic, reusable, and maintainable UI behavior.

Types of Directives

Angular provides three major types of directives.

🧩

Component Directives

Components are actually directives with templates. Every Angular component is considered a directive.

@Component()
🏗️

Structural Directives

Structural directives change the structure of the DOM by adding or removing elements.

*ngIf *ngFor *ngSwitch
🎨

Attribute Directives

Attribute directives change the appearance or behavior of elements without changing layout structure.

ngClass ngStyle

What does the * in *ngIf or *ngFor signify?

Understanding Angular structural directive syntax.

The Asterisk (*)

The * is shorthand syntax for a structural directive.

Angular transforms the element into an <ng-template> behind the scenes.

Structural directives change the structure of the DOM by adding or removing elements dynamically.

Angular Transformation

  Hello Angular
</div>

  </div>

How to Create a Custom Directive

Custom directives allow us to create reusable behavior and DOM interactions in Angular applications.

1

Generate the Directive 

ng generate directive highlight
2

Directive Code 

import {
  Directive,
  ElementRef,
  HostListener,
  Renderer2
} from '@angular/core';

@Directive({
  selector: '[appHighlight]'
})
export class HighlightDirective {

  constructor(
    private el: ElementRef,
    private renderer: Renderer2
  ) {}

  @HostListener('mouseenter')
  onMouseEnter() {
    this.renderer.setStyle(
      this.el.nativeElement,
      'backgroundColor',
      'yellow'
    );
  }

  @HostListener('mouseleave')
  onMouseLeave() {
    this.renderer.removeStyle(
      this.el.nativeElement,
      'backgroundColor'
    );
  }
}
3

Use the Directive 

<h1 appHighlight>
  Hover over me!
</h1>

Explanation of Key Parts

@Directive()

Marks the class as a directive and defines how Angular should use it in templates.

ElementRef

Gives access to the host DOM element.

Renderer2

Safely manipulates the DOM and is preferred over direct native element access.

@HostListener

Listens to events on the host element such as mouseenter or mouseleave.

Interview Questions

Important directive-related interview concepts.

What is the difference between a component and a directive?

Components include templates and control UI sections, while directives only add behavior to existing elements.

Why use Renderer2 instead of direct DOM manipulation?

Renderer2 provides a safer and platform-independent way to manipulate the DOM in Angular applications.

Can multiple structural directives be used on one element?

No. Angular allows only one structural directive per element. Use an additional wrapper like <ng-container> if needed.
