Angular Data Binding

Data Binding is the connection between the component (class) and the template (view). It allows data to flow from the component to the view, from the view to the component, or both.

Types of Data Binding in Angular

Angular provides four major types of data binding.

Binding Type Syntax Direction Use Case Example
Interpolation {{ '{{ expression }}' }} Component → View Display dynamic values Welcome, {{ '{{ user.name }}' }}
Property Binding [property]="expression" Component → View Set element properties <img [src]="imageUrl">
Event Binding (event)="handler()" View → Component Respond to user events <button (click)="save()">
Two-Way Binding [(ngModel)]="property" Component ↔ View Sync form inputs <input [(ngModel)]="username">
🔄

1. Interpolation ({{ '{{ }}' }})

Displays dynamic data inside templates.

Key Features

  • ✔️ Supports expressions
  • ✔️ Auto-updates UI
  • ✔️ Escapes HTML for security
  • ✔️ Supports Angular Pipes
Example 
// Component
export class UserProfileComponent {
  user = {
    name: 'Alex',
    score: 95
  };
}

Interview Question

What happens if you put an assignment inside interpolation?

Angular throws an error because interpolation only supports expressions, not statements like assignments.

❌ {{ '{{ x = 5 }}' }} is invalid

2. Property Binding ([ ])

Property binding sets values for HTML element properties, directives, and child component inputs.

Common Use Cases

  • ✔️ Image source binding
  • ✔️ Enable/disable buttons
  • ✔️ Pass data to child components
  • ✔️ Dynamic styles & classes
// Component
export class DashboardComponent {
  chartOptions = {
    responsive: true
  };

  isAdmin = true;
}

<app-chart [options]="chartOptions">
</app-chart>

<button [disabled]="!isAdmin">
  Delete Account
</button>

Interview Question

When would you use attribute binding vs property binding?

Use [attr.some-attr] when setting HTML attributes, and use property binding for DOM properties.

[attr.aria-label] vs [value]

3. Event Binding (( ))

Event binding listens for user interactions and triggers component methods.

Common Events

click (click)="save()"
keyup (keyup.enter)="search()" 
// Component
export class SearchComponent {

  searchTerm = '';

  onSearch() {
    // API call
  }
}

<input
  (keyup.enter)="onSearch()"
  [(ngModel)]="searchTerm"
/>

Interview Question

How would you prevent multiple button clicks? 

<button
  (click)="save()"
  [disabled]="isSaving"
>
  Save
</button>

isSaving = false;

save() {
  this.isSaving = true;

  this.service.save()
    .finally(() => {
      this.isSaving = false;
    });
}

4. Two-Way Binding

Synchronizes data between component and template instantly.

Key Concepts

  • ✔️ Combines property + event binding
  • ✔️ Keeps form values synchronized
  • ✔️ Uses ngModel
  • ✔️ Requires FormsModule
import { FormsModule }
from '@angular/forms';

<input
  [(ngModel)]="username"
/>

<app-search
  [(query)]="searchText"
>
</app-search>

Under the Hood of ngModel 

<input
  [ngModel]="value"
  (ngModelChange)="value = $event"
/>

5. Performance Considerations

Optimizing data binding improves Angular application performance.

Change Detection Strategies

  • ✔️ Default Strategy
  • ✔️ OnPush Strategy
  • ✔️ Async Pipe Optimization
  • ✔️ trackBy with ngFor
@Component({
  changeDetection:
    ChangeDetectionStrategy.OnPush
})

Important Interview Questions

Difference between (click) and onclick?

(click) is Angular’s event binding system integrated with change detection, while onclick is native DOM behavior.

Why avoid complex logic in templates?

Complex expressions run during every change detection cycle and can negatively affect performance.

Template Reference Variable vs Two-Way Binding?

Template reference variables are useful for one-time DOM access, while two-way binding is used for continuous synchronization between forms and component state.
