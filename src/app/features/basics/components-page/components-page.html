Angular Basics

Angular Components In-Depth Guide

Components are the core building blocks of Angular applications. Every Angular UI is composed of reusable, isolated, and maintainable components that work together to create dynamic applications.

🧩

1. What is an Angular Component?

Components control a portion of the user interface.

TypeScript Class

Contains the business logic, properties, methods, dependency injection, and state management.

HTML Template

Defines the structure and UI of the component using Angular template syntax.

CSS / SCSS

Styles the component and controls visual appearance and responsiveness.

Basic Component Structure

A component combines logic, template, and styling into a reusable UI block.

TypeScript 
import { Component } from '@angular/core';

@Component({
  selector: 'app-user-profile',
  templateUrl: './user-profile.component.html',
  styleUrls: ['./user-profile.component.css']
})
export class UserProfileComponent {
  userName = 'John Doe';
}

2. Component Lifecycle Hooks

Angular lifecycle hooks are methods called automatically by Angular during different phases of a component’s lifecycle.

Hook Purpose
constructor() Dependency injection and basic class initialization.
ngOnInit() Runs after Angular initializes component inputs. Commonly used for API calls and initialization logic.
ngOnChanges() Detects changes to input properties.
ngAfterViewInit() Runs after component view and child views are initialized. Useful for DOM access and ViewChild interactions.
ngOnDestroy() Cleanup logic such as unsubscribing from observables.

3. Smart vs Dumb Components

A very important architectural concept in Angular applications.

🧠

Smart Components

  • ✔️ Manage state & business logic
  • ✔️ Fetch data from APIs
  • ✔️ Handle user interactions
  • ✔️ Usually page-level components
🎨

Dumb Components

  • ✔️ Display UI only
  • ✔️ Receive data using @Input()
  • ✔️ Emit events using @Output()
  • ✔️ Highly reusable components

Interview Questions

Important Angular component interview concepts you should know.

1. What is the purpose of the @Component decorator?

The @Component decorator marks a TypeScript class as an Angular component and provides metadata that tells Angular how the component should behave, render, and be used.

  • ✔️ Defines selector
  • ✔️ Connects template
  • ✔️ Applies styles
  • ✔️ Configures encapsulation and animations

2. Difference between a Component and a Directive?

Components control a portion of the UI and always have a template, while directives are used to add behavior to existing DOM elements.

Example: *ngIf, *ngFor, and custom attribute directives.

3. When would you use ngAfterViewInit instead of ngOnInit?

Use ngAfterViewInit() when you need access to the DOM, child components, or @ViewChild references because the component view must be fully initialized first.

4. Difference between constructor(), ngOnInit(), and ngAfterViewInit()?

constructor()

Used for dependency injection and basic class initialization. It is a TypeScript feature, not Angular-specific.

ngOnInit()

Runs after Angular initializes input properties. Used for initialization logic and API calls.

ngAfterViewInit()

Runs after the component view and child views are initialized. Used for DOM manipulation and ViewChild access.
